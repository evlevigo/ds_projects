# Модель машинного обучения управления рисками при закупочных процедурах для молочного хозяйства.

**Описание проекта**

В рамках сотрудничества с IT-компанией, которая специализируется на выполнении проектов по машинному обучению, была проведена работа над проектом для фермерского хозяйства. В условиях контракта фермеру очень важно тщательно отобрать коров. Он определяет качество молока по строгим стандартам и желает соблюсти свой план развития молочного хозяйства. Фермер хочет, чтобы каждая бурёнка приносила не менее 6000 килограммов молока в год и чтобы её молоко было вкусным, соответствующим его критериям качества. Однако продавцы и технологи могут пытаться приукрасить характеристики своих животных.

Поэтому фермер просил меня разработать модель машинного обучения, которая поможет ему управлять рисками и принимать объективные решения о покупке. «ЭкоФерма» готова предоставить подробные данные о своих коровах. Мне необходимо было создать две прогнозные модели для отбора бурёнок в поголовье:

- Первая модель будет прогнозировать возможный удой коровы (целевой признак - Удой);
- Вторая модель будет рассчитывать вероятность получения вкусного молока от коровы (целевой признак - Вкус молока).

С помощью этих моделей необходимо отобрать коров по двум критериям:

- Средний удой за год должен быть не менее 6000 килограммов;
- Молоко должно быть вкусным.

В качестве исходных данных используются следующие файлы:

- ferma_main.csv
- ferma_dad.csv
- cow_buy.csv

В рамках работы над проектом я также ответил на вопросы заказчика:

**Сколько коров он может купить из стада на продажу с минимальным риском для себя? Почему?**

- Фермер может купить 4 коровы. Для остальных коров есть высокий риск, что молоко будет не вкусным.

**Какие ещё рекомендации можно ему дать?**

- Рекомендуется собрать как можно больше данных для построения более точной модели.

**Какие способы улучшения качества регрессионной модели вы использовали в этой задаче?**

- Для улучшения регрессионной модели были добавлены новые признаки, а также модифицированы существующие. Это позволило значительно улучшить качество модели.

**Возможно ли ещё улучшить результаты регрессионной модели? Если можно, то как? Если нет, то почему?**

Возможности для улучшения результатов:

- Расширить датасет. Выборка не очень большая, для лучших результатов следует увеличить её.
- Добавить новые признаки. На этапе подготовки данных часто проводят конструирование признаков — добавляют в датасет новые искусственные признаки, которые выделяют из других.
- Лучше подобрать гиперпараметры модели. Каждую модель можно тонко настроить, если знать, за какие «рычаги» потянуть. С их помощью можно улучшить модель.

**Какие метрики моделей классификации, в том числе логистической регрессии, важны при анализе рисков или экономических расчётов?**

Ошибки классификации включают в себя оценки двух видов: False Positive и False Negative (ошибки первого рода и второго рода) В задаче классификации молока ошибка первого рода — классифицировать корову со вкусным молоком как с не вкусным, а ошибка второго рода — определить корову с не вкусным молоком как со вкусным.

Precision оценивает точность, с которой модель присваивает объектам класс 1. Она реагирует на ошибки первого рода, однако не реагирует на ошибки второго.

Recall измеряет, смогла ли модель классификации присвоить класс 1 всем объектам этого класса. Её «слепое пятно» противоположно precision: она сосредоточена на ошибках второго рода и никак не учитывает ошибки первого.

Также необходимо правильно подобрать порог threshold, чтобы обеспечить наилучшие результаты.
